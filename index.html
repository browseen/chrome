Introduction

Browseen is a Chrome extension that enables universal web interaction through semantic commands rather than being tied to specific HTML structures. It allows you to perform common web actions like clicking, typing, scrolling, and even social interactions across any website using simple, intuitive commands.

The key benefits of Browseen include:

  • Platform Agnostic: Works across different websites with the same commands
  • Resilient to Changes: Not tied to specific CSS selectors or HTML structures
  • Accessibility-Focused: Designed for voice commands and assistive technologies
  • Intent-Based: Commands express what you want to accomplish, not how

Installation

  1. Download or clone the Browseen repository from GitHub
  2. Open Chrome and navigate to chrome://extensions/
  3. Enable "Developer mode" using the toggle in the top-right corner
  4. Click "Load unpacked" and select the Browseen extension folder
  5. The Browseen icon should now appear in your Chrome toolbar
Note: Once Browseen is officially released, you'll be able to install it directly from the Chrome Web Store.

Command Reference

Browseen uses a simple syntax for commands:

ACTION TARGET [PARAMETERS]

Navigation Commands

Command Description Example
CLICK Click on an element containing specific text CLICK 'Sign in'
GO TO Navigate to a URL or section of the page GO TO 'Settings'
SCROLL Scroll in a specified direction SCROLL DOWN 500
FIND Locate and highlight an element FIND 'Login'

Input Commands

Command Description Example
TYPE Type text into an input field TYPE 'Hello world' IN 'Search'
TOGGLE Toggle a checkbox, switch, or similar element TOGGLE 'Remember me'

Social Media Commands

Command Description Example
LIKE Like a social media post LIKE post CONTAINING 'news'
COMMENT Add a comment to a social media post COMMENT 'Great post!' ON first post

Command Examples

CLICK 'Sign in'

Clicks on a button, link, or other element containing the text "Sign in".

Browseen will search for elements with matching text content, aria-labels, or other attributes that might indicate a "Sign in" button or link.
TYPE 'Hello world' IN 'Search'

Types "Hello world" into a field labeled or placeholder with "Search".

This will find an input field associated with the text "Search" (through labels, placeholders, or nearby text) and type the specified text character by character.
SCROLL DOWN

Scrolls the page down by a default amount.

You can specify an amount by adding a number after the direction, e.g., SCROLL DOWN 500 to scroll down 500 pixels.
TOGGLE 'Accept cookies'

Toggles a checkbox or switch labeled "Accept cookies".

This works with checkboxes, radio buttons, and other toggle-like elements.
LIKE post CONTAINING 'news'

Finds a social media post containing the word "news" and likes it.

Works across various social media platforms including LinkedIn, Facebook, Twitter, etc.
COMMENT 'Great post!' ON first post

Adds the comment "Great post!" to the first social media post on the page.

You can also target specific posts with COMMENT 'text' ON post CONTAINING 'keyword' or COMMENT 'text' ON nearest post.

Advanced Usage

Element Targeting

Browseen uses multiple strategies to find elements based on your description:

  • Text Content: Elements with exact or partial text match
  • Attributes: Elements with matching aria-label, placeholder, title, etc.
  • Labels: Input fields with associated labels
  • Roles: Elements with specific ARIA roles
  • Proximity: Elements near the current viewport position

Context Menu Integration

Browseen adds items to Chrome's right-click context menu for quick access to common commands:

  • SCROLL DOWN: Quickly scroll down the page
  • CLICK selected text: When text is selected, click an element with that text
  • LIKE nearest post: Like the post closest to your current view
  • COMMENT on nearest post: Add a comment to the post closest to your current view

Command History

Browseen keeps track of your command history, making it easy to repeat previous commands. This history is accessible through the extension popup.

Important: Browseen doesn't store any sensitive information entered through the TYPE command. Your passwords and personal data remain secure.

Troubleshooting

Common Issues

Issue Solution
Command doesn't find the right element Try being more specific with your target text or use quotes for exact matching
Social media commands don't work Make sure you're on a page with social posts and try using "nearest post" if specific targeting fails
TYPE command doesn't enter text correctly Some websites use custom input fields; try different targeting methods or be more specific
Commands don't work on certain websites Some sites with heavy JavaScript or custom UI frameworks may require more specific commands

Debugging Tips

  1. Use the FIND command to check if Browseen can locate an element before trying to interact with it
  2. Check that the element is visible on screen; Browseen can't interact with hidden elements
  3. Try different variations of the target text to improve element matching
  4. For complex websites, break down operations into multiple commands
